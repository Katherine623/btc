# SMART 與強化學習應用於停車推薦系統

## 一、研究動機

在都市生活中，駕駛人在尋找停車位時，常常同時面臨多種考量，例如停車場距離目的地的遠近、進場等待時間、收費高低，以及是否仍有空位。這些因素彼此之間存在取捨關係，因此「最佳停車場」並不是單一條件即可決定，而是一個需要綜合判斷的決策問題。

本專題希望將日常生活中的停車選擇情境，轉化為一個可被人工智慧學習的決策任務。透過強化學習方法，系統可以在多次模擬中逐漸學會：當面對不同停車場條件組合時，應該推薦哪一個選項，才能獲得較高的整體效益。

此外，為了讓專題展示更具互動性與可視化效果，本研究也使用 Streamlit 建立操作介面，使使用者能即時調整參數、訓練模型、輸入案例並查看推薦結果，提升專題的展示完整度與實用性。

## 二、研究方法

本系統以「停車場推薦」作為應用情境，將每一個停車場抽象化為四項屬性：

1. 距離（Distance）
2. 等待時間（Wait Time）
3. 價格（Price）
4. 剩餘車位（Available）

系統每次會產生多個停車場候選方案，使用者的任務是從中選出最值得推薦的一個。為了讓模型能學習這個決策過程，本研究將每一次找停車位視為一次 episode，並使用 reward 來衡量該次選擇的好壞。

在方法設計上，本專題採用簡化版的 Q-learning：

1. 先為每個停車場建立狀態資訊。
2. 將連續數值特徵簡化離散化，降低狀態空間複雜度。
3. 由 agent 根據目前 state 選擇 action，也就是選擇某一個停車場。
4. 根據 reward 公式回饋該選擇的優劣。
5. 持續更新 Q-table，使 agent 在多次訓練後能學到較佳策略。

此方法的優點是邏輯清楚、容易展示，適合作為課堂專題的基礎模型；同時又保有強化學習中「互動、學習、逐步改進」的核心概念。

## 三、SMART 架構對應

本研究同時結合 SMART 目標設定法，讓專題主題更具有明確性與完整性。

### 1. Specific

研究目標明確設定為：建立一套能夠根據停車場條件，自動推薦較佳停車場的系統。

### 2. Measurable

系統使用 reward 作為量化依據，將停車決策的優劣轉換成可比較的數值。距離越短、等待越少、價格越低、空位越多，reward 越高；若停車場已無空位，則給予額外懲罰。

### 3. Achievable

本專題先從少量停車場、簡化版環境與 Q-learning 著手，不直接採用高維度或深度神經網路模型，使專題在課程時間與實作能力範圍內可順利完成。

### 4. Relevant

停車推薦屬於日常生活中非常具代表性的智慧決策問題，也與智慧交通、智慧城市、位置型服務等議題高度相關，因此具有良好的應用價值。

### 5. Time-bound

本系統將每一次停車選擇視為一個 episode，在有限回合內訓練 agent，並觀察 reward 是否隨訓練逐步改善，符合可在時間範圍內完成與評估的要求。

## 四、強化學習設計

### 1. 環境設計

本研究建立 `ParkingEnv` 作為停車推薦環境。每個停車場具有四項特徵：

- 距離：數值越小越好
- 等待時間：數值越小越好
- 價格：數值越小越好
- 剩餘車位：數值越大越好

系統可隨機產生測試案例，也可在 Streamlit 介面中由使用者手動輸入案例，方便展示不同情境下的模型反應。

### 2. 狀態表示

由於 Q-learning 不適合直接處理過多連續型狀態，本研究先將每個停車場的綜合分數離散化。分數計算方式如下：

```text
score = distance + wait_time + price - available
```

再依照 score 將停車場品質分成三類：

- 0：good
- 1：medium
- 2：bad

最後將所有停車場的離散結果組成 tuple，作為 Q-table 中的 state。

### 3. 動作設計

agent 的 action 即為「選擇某一個停車場」。如果系統中有 5 個停車場，則 action space 為 0 到 4。

### 4. Reward 設計

本專題的 reward 函數如下：

```text
reward = -2.0 * distance - 1.5 * wait_time - 1.0 * price + 2.5 * available
if available == 0:
    reward -= 30
```

此設計代表：

- 距離越近越好
- 等待時間越短越好
- 價格越低越好
- 可用車位越多越好
- 若完全沒有車位，則大幅扣分

因此，reward 直接反映了停車決策的整體品質，也讓系統能以數值方式學習推薦策略。

### 5. Q-learning 更新方式

本研究使用 Q-learning 進行學習，透過下列概念進行更新：

- `Q(state, action)`：在某個 state 下選擇某個 action 的價值
- `alpha`：學習率，控制新資訊影響程度
- `gamma`：折扣因子，控制未來報酬的重要性
- `epsilon`：探索率，控制隨機探索與既有知識利用的平衡

在訓練初期，系統透過較高的 epsilon 增加探索；隨著訓練進行，再逐步衰減 epsilon，使模型更傾向選擇目前認為較佳的行動。

## 五、Streamlit 系統展示方式

本專題除了核心演算法外，也透過 Streamlit 建立互動介面，主要功能如下：

1. 可調整停車場數量與訓練回合數。
2. 可調整 `alpha`、`gamma`、`epsilon` 等 Q-learning 參數。
3. 可產生隨機案例，觀察模型在不同資料下的推薦結果。
4. 可手動輸入每個停車場的條件，作為展示或比較案例。
5. 可顯示 RL 推薦停車場與真實 reward 最佳停車場是否一致。
6. 可顯示訓練 reward 曲線與 Q-table 預覽，幫助理解模型學習情形。

因此，Streamlit 版本不只是展示結果，也能讓使用者從互動操作中更容易理解整個強化學習流程。

## 六、實驗結果與分析

在本專案中，經過多回合訓練後，系統通常能逐漸學到較合理的停車推薦策略，整體 reward 也會比訓練初期穩定。透過 Streamlit 介面中的訓練摘要，可以觀察：

1. 平均 reward
2. 最近 100 回合平均 reward
3. 訓練過程中的最高與最低 reward
4. reward 隨 episode 變化的折線圖

從結果來看，模型已能在多數情況下選出不錯的停車場，尤其能辨識「空位多、距離較近、等待較短」這類 reward 較高的選項。

不過，本研究也觀察到：RL 推薦結果不一定每次都與真實最佳停車場完全一致。造成此現象的原因包括：

1. 狀態離散化後，部分不同案例可能落在相同 state，導致資訊損失。
2. 本模型採用簡化版 Q-learning，尚未納入更細緻的偏好權重或動態環境變化。
3. 每次 episode 只有一步決策，屬於簡化型的單步選擇問題。

因此，本專題的結果可視為「教學型、原理展示型」成果：它已成功呈現強化學習如何應用於停車推薦，但仍有進一步提升空間。

## 七、結論

本專題成功將 SMART 目標設定法與強化學習結合，完成一套具互動展示能力的停車推薦系統。透過本研究可以看出：

1. 停車推薦問題可以有效轉化為強化學習中的 state、action、reward 設計。
2. 使用 Q-learning 即可在簡化環境中學到基本決策策略。
3. 透過 Streamlit 介面，能更直觀地展示訓練過程、推薦結果與模型限制。
4. SMART 架構可幫助研究主題更具體、可衡量、可完成且具有實務相關性。

整體而言，本專題的價值在於：不只完成一個可執行的 AI 推薦系統，也用清楚的方式展示強化學習在真實生活情境中的應用可能性。

未來若要進一步擴充，可朝以下方向發展：

1. 納入使用者個人偏好，例如更重視價格或更重視距離。
2. 模擬即時停車位變動，讓環境更接近真實世界。
3. 導入更多資料特徵，例如時段、地區、天氣或交通狀況。
4. 改用 DQN、PPO 等深度強化學習方法，提高對複雜狀態的處理能力。
5. 結合地圖視覺化與定位服務，提升系統實用性。

## 八、摘要

本研究以停車推薦為應用場景，結合 SMART 與 Q-learning 建立一套可互動操作的 Streamlit 系統。模型根據距離、等待時間、價格與剩餘車位設計 reward，透過多次 episode 訓練 agent 學習停車決策策略。結果顯示，系統能在多數情況下提供合理推薦，並藉由視覺化介面清楚呈現訓練與決策過程。雖然目前仍屬簡化模型，但已具備作為課堂專題展示與強化學習入門案例的完整性。
